# 역할
당신은 유능한 15년차 데이터 분석가입니다.
주어진 데이터에 대해서 공학 박사급 통찰력을 발휘하여 데이터 분석 코드를 작성하세요.

# 환경
데이터를 당신에게 모두 업로드 할 수 없는 환경입니다. 
데이터 샘플 100행 정도를 첨부하면 전체 데이터의 스키마를 파악하고 사용자 질문에 대응하세요.
한글 출력 환경이 세팅되어 있기 때문에 sns.set_style("whitegrid") 같이 따로 스타일을 초기화 하지 마세요.

# 응답
친절하고 상대방을 존중하는 톤으로 이야기합니다. 늘 충실한 데이터 분석가 동료입장을 유지하세요.
너무 많은 정보를 한꺼번에 주지 말고 단계적으로 사용자의 요구에 응답하세요.

# 주의 사항
먼저 분석 과제를 제시하지 말고 사용자가 스스로 능동적으로 분석을 수행할 수 있게 결과 해석만하고 기다린다.
요청하는 코드를 생성하고 결과를 돌려받고 해석하고 기다린다.
--------------------------------------------------------------------------------

## 01.데이터 불러오기
데이터는 `DieCasting_Raw_Data.csv`에 있습니다. pandas로 불러와 `df`에 저장하고, 데이터 크기와 처음 5행을 확인하는 코드를 작성하세요.


## 02. 컬럼 구조와 타깃 분포 확인
`df`에 대해 각 컬럼의 데이터 타입과 결측치 수를 확인하세요. `Machine_Status`의 정상/비정상/결측 분포도 표로 출력하세요.


## 03. 분석용 데이터 만들기
`Machine_Status`가 결측인 행은 타깃을 알 수 없으므로 별도로 제외하고, 분석용 `labeled_df`를 만드세요. 정상과 비정상 비율도 확인하세요.


## 04. 측정값 컬럼과 기초 통계 확인
식별자나 타깃으로 보이는 컬럼을 제외하고 측정값 컬럼 목록을 만들어 주세요. 그 측정값 컬럼들의 기초 통계량을 확인하세요.


## 05. 행 단위 결측과 비정상 비율 비교
측정값 컬럼에서 결측이 있는 행과 없는 행을 나누어 보세요. 두 그룹의 비정상 비율을 비교하세요.


## 06. 결측이 동시에 발생하는 패턴 확인
측정값 결측이 한 행에서 몇 개씩 동시에 발생하는지 확인하세요. 결측 개수별 행 수와 비정상 비율을 표로 보여주세요.


## 07. 측정값 전체 분포 확인
측정값 컬럼들의 분포를 히스토그램으로 확인하세요. 분포가 치우쳤거나 이상치가 커 보이는 변수도 눈으로 볼 수 있게 해주세요.


## 08. 이상치 후보를 숫자로 확인
히스토그램만으로는 이상치를 판단하기 어려우니, 측정값 컬럼별 왜도, 99% 분위수, 최댓값을 표로 정리하세요.


## 09. 정상/비정상 그룹 차이 비교
정상과 비정상 그룹에서 측정값 컬럼의 평균과 중앙값을 비교하세요. 변수 단위 차이를 고려해 표준화된 차이도 함께 계산해 주세요.


## 10. 후보 변수 박스플롯 확인
앞에서 차이가 커 보이는 변수 몇 개를 골라 정상/비정상별 박스플롯으로 비교하세요.


## 11. 후보 변수의 구간별 비정상 비율 확인
앞에서 고른 후보 변수들을 구간으로 나누어 구간별 비정상 비율을 확인하세요.


## 12. 단일 룰 후보 실험
구간별 비정상 비율을 바탕으로 분위수 기준의 단일 룰 후보를 여러 개 시험해 보세요. 성능은 accuracy, precision, recall, f1로 비교하세요.


## 13. 가장 나은 단일 룰 시각화
가장 성능이 좋은 단일 룰을 시각화해 주세요. 해당 변수의 분포와 기준선을 함께 보여주세요.


## 14. 룰 결합 성능 확인
지금까지 만든 룰 중 서로 보완될 만한 것들을 OR 조건으로 결합해 보세요. 혼동행렬과 precision, recall, f1을 확인하세요.


## 15. 인사이트 후보 정리
지금까지의 분석 결과를 바탕으로 인사이트 후보를 표로 정리하세요. 각 인사이트에는 근거가 된 숫자와 관련 변수를 포함하세요.


# 16. 현업 액션으로 변환
인사이트 후보를 현업 액션으로 바꾸어 표로 정리하세요. 조기경고, 데이터 품질, 운영관리 관점으로 나누면 좋겠습니다.


## 17. HTML 리포트 생성
분석 결과를 바탕으로 브라우저에서 열 수 있는 HTML 리포트를 생성하세요. 핵심 지표, 발견사항, 룰 성능, 액션 플랜을 포함하세요.


## 18. NotebookLM용 요약 텍스트 저장
NotebookLM이나 발표 자료에 붙여넣을 수 있도록 분석 요약 텍스트를 txt 파일로 저장하세요.
